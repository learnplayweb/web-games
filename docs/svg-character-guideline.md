# 캐릭터 SVG 제작 가이드

## 목적

Web Games의 캐릭터 파츠 SVG를 제작하기 위한 기준 문서이다.

모든 SVG는 본 문서를 기준으로 작성한다.

문서에 없는 기능이나 구조는 임의로 추가하지 않는다.

---

# 기본 컨셉

* 단순하고 귀여운 디자인
* 둥글고 통통한 느낌
* 날카로운 모서리 사용 금지
* 약간 찌글찌글한 실루엣 허용
* 어린이가 좋아할 수 있는 디자인
* 작은 화면에서도 형태를 쉽게 구분할 수 있어야 한다.
* 선은 크레파스나 분필 질감을 가지며 부드러운 느낌을 위해 굵게 표현

---

# 캐릭터 구조

캐릭터는 최대 3개의 파츠를 사용한다.

1단계

* 머리

2단계

* 머리
* 몸(팔 포함)

3단계

* 머리
* 몸(팔 포함)
* 다리

모든 파츠는 동일한 기본 도형을 사용한다.

---

# 얼굴

머리에는 항상 얼굴을 표시한다.

구성

* 눈 : 작은 점 2개
* 입 : 짧은 선 1개

표정은 단순하게 유지한다.

---

# 팔 / 다리

팔과 다리는

* 선 하나
* 끝에 작은 원

형태를 사용한다.

너무 길거나 복잡하게 만들지 않는다.

---

# 크기 비율

머리

* 가장 큼

몸

* 머리의 약 30~50%

다리

* 몸과 비슷한 크기

---

# 색상

기본 SVG는 단색으로 제작한다.

색상 시스템은 별도로 적용한다.

SVG에는 특정 색상에 의존하는 효과를 넣지 않는다.

---

# 애니메이션 대응

SVG는 애니메이션을 고려하여 제작한다.

각 파츠를 독립적으로 선택할 수 있도록 적절한 id 또는 그룹(<g>)을 사용한다.

예)

* head
* body
* legs

---

# 애니메이션 방식

SVG 내부에서 애니메이션를 구현하지 않는다.

JavaScript 또는 CSS에서 애니메이션을 적용한다.

기본 애니메이션

* 둥실둥실(상하 이동)
* 살랑살랑(좌우 기울기)

추가 애니메이션

* 머리, 몸, 다리가 서로 약간 다른 각도로 흔들림
* 각 파츠는 약간의 시간차를 두고 움직임
* 눈 깜빡임(랜덤 간격)

SVG는 위 애니메이션을 적용하기 쉬운 구조로 작성한다.

---

# 성능

* SVG는 최대한 가볍게 작성한다.
* 불필요한 path 사용 금지.
* 가능하면 circle, ellipse, rect 등을 적극 활용한다.
* path 수는 최소화한다.

---

# 스타일

* SVG 하나만 보아도 귀엽게 보여야 한다.
* 어떤 파츠끼리 조합되어도 어색하지 않아야 한다.
* 단순함을 최우선으로 한다.
* 과한 디테일은 사용하지 않는다.

---

# 구현 원칙

* 재사용 가능한 구조
* 반응형(viewBox 사용)
* JavaScript에서 쉽게 접근 가능한 구조
* 유지보수가 쉬운 구조
