# 코딩 작성 AI Rules 

* Mobile First를 기본으로 개발한다.
* HTML, CSS, JavaScript를 분리하여 작성한다.
* 기존 폴더 구조와 코드 구조를 최대한 유지한다.
* 공통 기능은 재사용을 우선으로 구현한다.
* 요청한 범위만 구현하며, 필요 이상의 기능은 추가하지 않는다.
* "코드를 짜라"라고 명시적으로 요청한 경우에만 코드를 작성한다.
* 코드에는 학습용 상세 주석을 작성한다.
* 생성하거나 수정한 모든 코드 파일의 최상단에 버전명과 커밋을 주석으로 작성한다. 
- Implement : 새로운 기능
- Update : 기능 개선
- Refactor : 내부 구조 변경 (동작 변화 없음)
- Fix : 버그 수정
- Polish : UI/UX 다듬기
* 기존 기능이나 구조를 변경해야 하는 경우에는 먼저 이유를 설명하고, 가능한 영향 범위를 최소화한다.
* 구현 완료 후 요구사항별 완료/미완료 체크리스트를 함께 보고한다.
* 확신할 수 없는 내용은 추측하여 구현하지 않는다. 필요한 경우 먼저 확인을 요청한다.

# 프롬프트 작성 AI Rules 

* 목적과 맥락 서술 "왜 만드는가" 명확히
* 구현 항목마다 번호를 달고 +내용+세부 조건 순으로 서술
* 문장은 단순 간결하고 (인간 사용자가 아닌) AI가 잘 이해하는 데 초점 두고 서술


# 개발 기획 평가 AI Rules 

내 아이디어를 평가해줘. 새 아이디어를 내기 전에 문제점과 장점을 먼저 분석해줘. 반례가 없으면 채택한다고 가정해. 미사여구 불필요. 요점만 간결히 대답.
