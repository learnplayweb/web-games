# Reward System

## 목적

보상 시스템은 플레이어의 지속적인 학습과 도전을 유도하기 위해 설계한다.

기본 학습 보상은 유지하면서, 쉬운 단계 반복 플레이를 통한 과도한 Gold 획득은 제한한다.

최고 난이도는 반복 학습 자체를 장려하는 방향으로 운영한다.

---

## Gold

Gold는 프로젝트의 유일한 보상 재화이다.

모든 게임은 Gold를 공통으로 사용한다.

Gold는 캐릭터 성장 및 관련 콘텐츠에 사용한다.

---

## 일반 레벨 보상

### 문제 정답 보상

단계별 최고 별점에 따라 문제당 획득 Gold를 차등 적용한다.

| 저장된 최고 별점 | 문제당 Gold |
| -------- | -------: |
| 없음       |      +10 |
| ★        |       +7 |
| ★★       |       +4 |
| ★★★      |       +1 |

문제를 맞힐 때마다 저장된 최고 별점 기준 Gold를 즉시 지급한다.

---

### 콤보 보너스

단계 종료 시 최장 Combo를 기준으로 보너스를 지급한다.

```
Combo Bonus = maxCombo × COMBO_MULTIPLIER
```

단계별 최고 별점에 따라 COMBO_MULTIPLIER를 차등 적용한다.

| 이전 최고 별점 | COMBO_MULTIPLIER |
| -------- | ---------------: |
| 없음       |                4 |
| ★        |                3 |
| ★★       |                2 |
| ★★★      |                1 |

COMBO_MULTIPLIER는 상수로 관리하며, 밸런스 테스트 결과에 따라 쉽게 조정할 수 있도록 구현한다.


---

### 별점

정답률에 따라 별점을 부여한다.

| 정답률    | 별점  |
| ------ | --- |
| 100%   | ★★★ |
| 2/3 이상 | ★★  |
| 1/3 이상 | ★   |
| 1/3 미만 | 없음  |

---

### 별점 보상

별점의 가치는 다음과 같다.

| 별점  | Gold |
| --- | ---: |
| ★   |   10 |
| ★★  |   30 |
| ★★★ |   50 |

최초 도전 또는 최고 별점 갱신 시 지급 Gold는

```
새 최고 별 가치 − 이전 최고 별 가치
```

로 계산한다.

예)

* 없음 → ★ = +10
* ★ → ★★ = +20
* ★★ → ★★★ = +20
* ★ → ★★★ = +40

별점 유지 또는 하락 시 추가 Gold는 지급하지 않는다.

---

### 최고 별점 저장

최고 별점은 게임 전체가 아니라 **각 게임의 각 단계별로 독립 저장**한다.

예)

* Clock Lv.1 : ★★★
* Clock Lv.2 : ★★
* Number Connect Lv.3 : ★★★

---

## 최종 레벨 보상 규칙

### 목적

최고 난이도의 반복 학습을 장려한다.

---

### 문제 정답 보상

일반 레벨과 동일하다.

---

### 콤보 보너스

일반 레벨과 동일하다.

감소 규칙을 적용하지 않는다.

---

### 별점

매 플레이마다 새로 계산하여 최근 별점을 저장한다.

최고 별점은 저장하지 않는다.

---

### 별점 보상

매 플레이마다 지급한다.

| 별점  | Gold |
| --- | ---: |
| ★★★ |  +50 |
| ★★  |  +30 |
| ★   |  +10 |
| 없음  |   +0 |

---

## 구현 원칙

* 모든 게임은 동일한 보상 시스템을 사용한다.
* 게임별 예외 규칙은 가능한 만들지 않는다.
* 보상 수치는 상수로 관리하여 밸런스 조정을 쉽게 한다.
* Gold는 프로젝트 전체에서 하나의 재화만 사용한다.
* 일반 레벨의 보상 계산은 플레이 시작 시 저장된 최고 별점을 기준으로 하며, 플레이 도중에는 변경되지 않는다.
