# 프로젝트 파일 역할

## 목적

프로젝트의 파일 역할을 명확히 분리하여 유지보수와 확장을 쉽게 한다.

새로운 기능을 구현할 때는 기존 역할을 우선 유지하며, 하나의 파일에 여러 역할을 섞지 않는다.

---

# characters/

캐릭터 시스템 전체를 관리하는 폴더이다.

특정 게임에 종속되지 않는다.

모든 게임이 공통으로 사용한다.

---

## character-shop.html

캐릭터 상점 화면

역할

* 상점 UI
* 파츠 구매 화면
* 색상 화면
* 캐릭터 표시
* 이름 변경 UI
* 저장 UI

비즈니스 로직은 작성하지 않는다.

---

## character-shop.css

상점 화면 전용 스타일

역할

* 레이아웃
* 애니메이션
* 버튼
* 접기/펼치기 UI

---

## character-shop.js

상점 화면 전용 로직

역할

* 버튼 이벤트
* 화면 갱신
* 접기/펼치기
* 상점 화면 제어

공통 캐릭터 로직은 작성하지 않는다.

---

## characters.js

캐릭터 공통 시스템

역할

* 캐릭터 생성
* 조합
* 해체
* 이름 변경
* 색상 적용
* 캐릭터 렌더링
* 캐릭터 상태 관리

게임 화면에서도 동일한 코드를 사용한다.

---

## characterData.js

캐릭터 데이터만 관리한다.

예)

* 파츠 목록
* 색상 목록
* 꾸밈 목록
* 가격
* 상수
* 기본 설정

함수는 가능한 작성하지 않는다.

---

## assets/

캐릭터 리소스를 저장한다.

예)

* SVG
* 아이콘
* 이미지

---

# games/

각 게임 전용 폴더이다.

게임별 HTML, CSS, JavaScript를 관리한다.

캐릭터 공통 기능은 작성하지 않는다.

필요 시 characters 폴더의 공통 기능을 사용한다.

---

# data/

게임 전용 데이터만 관리한다.

예)

* 단계 정보
* 문제 생성 데이터
* 게임 설정

캐릭터 데이터는 저장하지 않는다.

---

# 구현 원칙

* 공통 기능은 characters 폴더에서 관리한다.
* 게임별 기능은 games 폴더에서 관리한다.
* 데이터와 로직을 분리한다.
* UI와 비즈니스 로직을 분리한다.
* 하나의 파일은 하나의 역할만 담당한다.
* 새로운 기능 추가 시 기존 파일 역할을 변경하지 않는다.
