상점의 파츠 선택 UI를 다음과 같이 수정한다.

1. 파츠 슬롯을 누르면 모달을 표시한다.

2. 모달에는 다음을 표시한다.
- 선택한 파츠를 크게 표시
- 구매 버튼
- 적용 버튼(랜덤 박스 제외)

3. 버튼 구성
- 일반 파츠
  - [구매 💎100]
  - [적용 💎10]
- 랜덤 박스
  - [구매 💎○○]

4. 구매
- 보유 여부와 관계없이 항상 구매할 수 있다.
- 동일한 파츠를 여러 개 구매할 수 있다.
- 이미 보유한 파츠도 추가 구매 가능하다.

5. 적용
- 이미 보유한 파츠만 활성화한다.
- 적용하면 현재 캐릭터의 **머리**를 해당 파츠로 변경한다.
- 현재 머리와 동일한 파츠인 경우에는 버튼 대신 '적용 중' 상태를 표시한다.

6. 모달 닫기
- 모달 바깥 배경을 누르면 닫는다.
- 별도의 닫기(X) 버튼은 만들지 않는다.

현재 단계에서는 UI만 구현한다.
구매, 적용, 골드 차감 등 실제 기능은 구현하지 않는다.

## 목표

상점에서 파츠를 구매하여 인벤토리에 저장하고, 보유 개수를 관리한다.

## 요구사항

### 1. 구매

- 일반 파츠 구매 시 💎100 차감
- 랜덤 박스 구매 시 💎70 차감
- 💎이 부족하면 구매하지 않는다.
- 동일한 파츠는 여러 번 구매할 수 있다.

### 2. 인벤토리

각 파츠의 보유 개수를 저장한다.

예)

Circle : 3
Triangle : 1
Square : 0

### 3. 화면 표시

상점 슬롯에는 현재 보유 개수를 배지 형태로 표시한다.

- 미보유 : 0
- 보유 : 실제 개수

### 4. 저장

구매 결과는 save.js를 통해 localStorage에 저장한다.

새로고침 후에도 보유 개수와 💎이 유지되어야 한다.

### 5. 제외 사항

아직 구현하지 않는다.

- 머리 적용
- 조합
- 해체
- 이름 변경
- 색상
- 꾸밈

## 구현 원칙

- 구매 로직과 저장 로직을 분리한다.
- 인벤토리는 객체 형태로 관리한다.
- 기존 저장 시스템과 호환되도록 구현한다.

변경된 파일만 출력한다.

구현 완료 후 요구사항별 완료/미완료 체크리스트를 함께 보고한다.
